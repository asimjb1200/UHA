I. Introduction[1] 

Endeavor Technologies, hereafter referred to as Endeavor Tech, has developed the following user manual to provide details on the proper use of the inventory and scheduling information system application designed for Alaska Ultimate High Adventure, hereafter referred to as UHA. This manual provides a walk-through of various processes found in the web application as of May 2019.

The system provides solutions to the issues identified in the previous system utilized by UHA. The following business problems are as follows:

●     Difficulty in client information ingestion and storage.

●     Lack of organizational management for individual treks.

●     Inability to create a standardized schedule of events for each individual trek, editable for on-the-fly changes.

●     Difficulty to track inventory location changes or damaged equipment for report generation.

●     Loss of information due to lack of standardized reports and employee tools.

 

 

 

A. System Overview

The information system has been integrated as a web application, giving UHA employees and program coordinators the ability to input, store, retrieve, search, and identify equipment inventory and scheduling information for clients. All data ingested by the system is stored in a secure cloud-based database environment and connected to a employee-facing web application. Upon proper login credentials by UHA staff, information can be accessed from the database in any location with internet connectivity bandwidth. The web application portal provides three levels of administrative privileges, including superuser, administrator, and basic user. With the various permission levels, access to sensitive information or inadvertent alteration of data is minimized. Therefore, administrator and user views will appear differently to ensure the confidentiality of certain sensitive information. The application satisfies the following objectives: create, edit, and delete client information; add, edit, update availability status, and delete inventory items; create, edit, and delete food items and menus; create, edit, and delete client trip details; and generate trip reports for administrative purposes.

Inventory is another primary page for the application. Here you can live track the full inventory for UHA.

 

The user can search for a supply in the inventory. They will need to insert the supply name and category in the supply name and category field. Also, they can add new supplies to the inventory. Moreover, they will be allowed to edit or remove existing inventory.

 

 

Hitting the remove button will take the user to the page above and ask for confirmation to delete the selected item.

 

 

 

Hitting the edit button will take the user the page where they can see information about inventory items and be able to edit them. They can edit the supply name, category, quantity and price. After that, the user will hit submit and changes will be saved and the user will be taken back to the inventory page.

 

Finally, hitting the add supply button will take the user to the page above. In this page, the user will be able to add a new supply to the inventory. The user will need to add a supplyName, category, quantity and price. After adding all that, the user will hit submit and the supply will be added.

5.    Food and Meals
Food and Meals shows the Meals and Menus as well as food items and where they are located.

The user can see all existing meals and menus. By clicking any meal or menu, the user will be able to edit and view information. The user will be taken to the pages below:

 

By clicking any of the existing meals, the user will be able to see information about that meal and edit it. The user can change the meals name, food items in that meal and add some description. After that, the user will hit submit to save changes or hit the green button on the right side of the page to go back to the top.

 

By hitting the add meal button, the user will be taking to the page above. This option will allow the user to build a new meal. The user will insert the name of the meal, choose food items for the meal and add description. After that, the user will hit the submit button to build the meal or hit the green button to go to the top of the page.

By clicking in any of the existing menus, the user can see and edit the name of the menu and what meals are in the menu. If the user make any changes to the menu, he will hit submit after that to save changes.

 

By clicking the add food button, the user will be taken to the page above. The user will be able to insert food name, price, quantity and warehouse. After that, the user will hit the submit button to add the food item.

If the user wants to delete any food item, it will take them to the confirmation page to delete that item.

 

 

6.    Storage (Equipment)
These are the storage locations as well as the Kayaks and Trailers. Storage holds the information for viewing the warehouses, kayaks, and the trailers. The user will also be able to edit them.

The Trailers can be edited by pressing the yellow edit button. From this page you can change the warehouse location of the trailer, the condition, and the availability.

The kayaks can be edited by pressing the yellow edit button. From this page you can change the warehouse location of the kayak, the condition, and the availability.
